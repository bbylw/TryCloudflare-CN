01 / 概览

什么是快速隧道（Quick Tunnels）？

开发者可以使用 TryCloudflare 工具来体验 Cloudflare Tunnel，而无需将站点添加到 Cloudflare 的 DNS。它会启动一个进程，在 trycloudflare.com 上生成一个随机子域名，并将发往该子域名的请求经由 Cloudflare 网络代理转发到你本地运行的 Web 服务器。

⚡

零配置

不需要拥有域名，也无需把 DNS 指向 Cloudflare 的 nameserver，开箱即用。

🌐

全球加速

流量经过 Cloudflare 边缘网络，自动享受其速度与安全层防护。

🔒

安全穿透

本地服务无需暴露公网端口，出站连接即可被外部安全访问。

🧪

为测试与开发而生

快速隧道是体验 Cloudflare Tunnel 的最佳入口 —— 想分享笔记本上的项目、做跨浏览器兼容测试，或从不同地区跑速度测试，都能一键搞定。若需用于生产，请创建远程托管隧道。

!

注意：快速隧道仅用于测试与开发。Cloudflare 不保证其 SLA 或在线率，且可能在该通道上试验新功能。请勿用它部署生产网站。

02 / 上手

三步启动你的第一条隧道

只要本地有一个可访问的服务，三步即可把它分享到任意网络中的设备。

1

安装 cloudflared

按官方安装说明安装命令行工具。若已有旧版本，请升级到 2020.5.1 或更高。

2

启动本地 Web 服务

在 localhost 上运行一个可供 cloudflared 连接的服务器，例如监听 http://localhost:8080 的服务。

3

运行隧道命令

cloudflared 会自动生成随机子域名并打印到终端，供你使用和分享。

cloudflared tunnel --url http://localhost:8080
i

提示：若 .cloudflared 目录中存在 config.yaml 配置文件，快速隧道暂不支持。请临时重命名该文件后再试。

04 / 边界

能力与限制

快速隧道为免费测试通道，以下限制仅适用于它。若需突破限制，请注册 Cloudflare 账户并创建托管的 Cloudflare Tunnel。

并发上限

200 个在途请求

快速隧道对同时代理的请求数有硬性上限，目前为 200。一旦触顶，HTTP 响应将返回 429 状态码。

协议支持

不支持 SSE

快速隧道不支持 Server-Sent Events（SSE），需要实时流式推送的场景请改用托管隧道。

!

升级路径：为避免上述限制，请注册 Cloudflare 账户，并创建 Cloudflare Tunnel 用于生产环境。

05 / FAQ

常见问题

TryCloudflare 有哪些使用场景？ +
  • 为笔记本上想分享给他人的项目创建一个 Web 服务器，对方可在不同网络中访问。
  • 创建免费隧道、在不同浏览器中测试链接，验证新站点的浏览器兼容性。
  • 使用 Pingdom 或 WebPageTest 等工具，连接到随机子域名，从不同地区进行速度测试。
为什么 Cloudflare 免费提供这项服务？ +
  • 希望更多用户体验 Cloudflare Tunnel 的速度与安全提升，试用后将其用于生产站点。
  • Cloudflare 以往功能要求拥有域名、配置 DNS 记录；希望让更多产品无需这些负担即可试用。
  • 不保证 SLA 或在线率，计划在免费隧道上测试新功能，为生产部署前提供测试连接组。免费隧道仅供测试与开发。
快速隧道何时会返回 429？ +
当并发在途请求数达到硬性上限（目前 200）时，新请求会被拒绝并返回 429 状态码。降低并发或改用托管隧道即可解决。
config.yaml 会影响快速隧道吗？ +
会。若 .cloudflared 目录中存在 config.yaml，快速隧道暂不支持该功能，请临时重命名该文件后再使用。
